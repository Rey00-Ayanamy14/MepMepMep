# Courier Management System · Frontend

SPA на React для работы с сервером `/Users/inostaziss/IdeaProjects/courier_management_system`. Интерфейс покрывает все основные роли и эндпоинты, которые доступны в бекенде:

- **Auth** — `POST /auth/login`
- **Admin** — CRUD пользователей (`/users`), машин (`/vehicles`), товаров (`/products`)
- **Manager** — просмотр / создание / редактирование доставок (`/deliveries`), массовая генерация (`/deliveries/generate`), расчет маршрута (`/routes/calculate`)
- **Courier** — список и детали назначенных доставок (`/courier/deliveries`)

## Быстрый старт

```bash
npm install
npm run dev
```

По умолчанию фронт ожидает работающий бэкенд на `http://localhost:8080`. При необходимости переопределите адрес через переменную `VITE_API_URL`:

```bash
VITE_API_URL=http://localhost:8081 npm run dev
```

Сборка production-версии:

```bash
npm run build
npm run preview
```

## Структура приложения

- `src/state/AuthContext` — хранение токена, данных пользователя и функция login/logout
- `src/api` — тонкая обертка над fetch с автоматической установкой `Authorization` и построением query-параметров
- `src/components/layout/AppLayout` — общий каркас (sidebar + header), защищенный роутинг и навигация по ролям
- `src/pages/*` — страницы для каждой роли:
  - `LoginPage` — форма авторизации
  - `DashboardPage` — быстрые показатели и виджеты по текущей роли
  - `UsersPage`, `VehiclesPage`, `ProductsPage` — CRUD-интерфейсы администратора
  - `DeliveriesPage` — рабочее место менеджера: фильтры, детали, ручное создание, массовая генерация и расчет маршрута
  - `CourierDeliveriesPage` — список и детализация задач курьера
- `src/components/deliveries/DeliveryDetails` — общий компонент отображения доставки (используется менеджером и курьером)

## Особенности

- Хранение токена и сведений о пользователе в `localStorage` с автоматическим редиректом после логина/логаута
- Фильтры по статусам, датам и курьерам для менеджера и курьера
- Динамические формы для маршрутов и товаров внутри доставки, подсказки по ID, если курьеры недоступны (только админ может получить их список)
- Обработка ошибок API (403/400) с понятными сообщениями
- Базовый набор адаптивных стилей без сторонних UI-библиотек

## Полезные подсказки

- Данные о том, какие действия доступны конкретной роли, подсвечиваются в сайдбаре — ссылки, не относящиеся к роли, скрыты
- Для генерации доставок можно начать с примера JSON прямо в интерфейсе и отредактировать его под нужные маршруты/даты
- Проверка маршрута (`/routes/calculate`) позволяет убедиться, что временные промежутки подходят, до сохранения доставки
